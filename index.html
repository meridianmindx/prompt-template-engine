Why use Prompt Template Engine?

Validated Variables

Ensure every template variable meets your constraints — min/max length, regex patterns, choice lists, or custom validators.

Safe Rendering

No more f-strings or format() — templates are rendered with explicit variable substitution only. No code execution, no surprises.

Reusable Templates

Define named templates with optional guard conditions. Register, retrieve, and render with one line of code.

Structured Output

Templates can validate output against JSON schemas, regex patterns, or choice lists. Get guaranteed well-formed output.

Zero Dependencies

Pure Python stdlib only. No pip install headaches, no version conflicts, no build steps required.

Test-Ready

Templates are deterministic given valid inputs. Easy to test, easy to mock, easy to integrate.

Examples

Basic Template (no guards)

from prompt_templates import TemplateEngine

engine = TemplateEngine()

engine.add_template("greeting",
    "Hello, {name}! Welcome to {place}."
)

result = engine.render("greeting", {
    "name": "Alice",
    "place": "Wonderland"
})
# Result: "Hello, Alice! Welcome to Wonderland."

Template with Guards

from prompt_templates import TemplateEngine, GuardCondition

engine = TemplateEngine()

engine.add_template("code_gen",
    "Write {type} code for {task}. "
    "Output must be valid {output_type} JSON.",
    guards={
        "type": GuardCondition(name="type",
            choices=["Python", "TypeScript", "Rust"]),
        "task": GuardCondition(name="task",
            min_length=10, max_length=500),
        "output_type": GuardCondition(name="output_type",
            choices=["String", "Integer", "Boolean"])
    }
)

result = engine.render("code_gen", {
    "type": "Python",
    "task": "Build a REST API with FastAPI",
    "output_type": "String"
})

Guard Conditions

from prompt_templates import GuardCondition

# String length constraints
GuardCondition(name="name", min_length=2, max_length=50)

# Regex pattern
GuardCondition(name="email", regex=r"^[^@]+@[^@]+\.[^@]+$")

# Choice list
GuardCondition(name="tone", choices=["formal", "casual", "technical"])

# Custom validator
GuardCondition(name="age", validator=lambda x: int(x) > 0)

Installation

# Install from PyPI
pip install prompt-templates

# Or clone and install locally
git clone https://github.com/meridianmindx/prompt-template-engine.git
cd prompt-template-engine
pip install -e .

Quick Start

from prompt_templates import TemplateEngine

# Create engine
engine = TemplateEngine()

# Add a template
engine.add_template("review",
    "Write a {tone} review of {product}."
)

# Render with variables
result = engine.render("review", {
    "tone": "professional",
    "product": "AI assistant"
})
